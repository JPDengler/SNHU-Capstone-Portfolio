Overview

The Nuoro Weight-Tracking App continues to evolve with Enhancement Two, focusing on Algorithms & Data Structures. This update improved how users interact with and organize their data by introducing sorting and filtering functionality, as well as optimizing database queries for efficiency. The enhancements aim to streamline user experiences and maintain data integrity, further building on the foundation established in Enhancement One.

Users can now sort entries by date and weight, both in ascending and descending orders, and filter entries using date and weight ranges. These new features ensure that users can quickly find and organize their data, no matter the size of their dataset.

Key Improvements

  • Sorting Options: Added functionality to sort weight entries by Date (Ascending/Descending) and Weight (Ascending/Descending).
  • Search Feature: Allowed users to filter entries based on a specific date range or weight range, providing quick access to relevant data.
  • Optimized Database Queries: Enhanced SQLite queries using ORDER BY and WHERE clauses to improve retrieval speeds.
  • Bug Fixes: Resolved issues with disappearing charts during sorting and fixed RecyclerView scrolling errors for large datasets.

Technical Implementation

The following changes were implemented to bring these features to life:

  • DatabaseHelper: Refactored to include new methods for sorting and searching weight entries using optimized SQL queries.
  • DataDisplayActivity: Enhanced UI components to support sorting and filtering functionality based on user input.
  • Dynamic Graph Updates: Refactored graphing logic to dynamically reflect sorting and filtering results.
  • Input Validation: Added checks to ensure users enter valid date ranges and weight thresholds for filtering operations.
  • Bug Fixes: Resolved RecyclerView scrolling and chart rendering issues, ensuring smooth user interactions.
