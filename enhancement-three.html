Overview

The Grazioso Salvare Dashboard is a data-driven web application designed for animal rescue organizations, providing tools for filtering, visualizing, and managing rescue animal data. Initially, the system faced challenges in query efficiency, error handling, security, and testing.

Enhancement Three focuses on improving the reliability, scalability, and security of the application by introducing optimized database queries, structured error management, secure credential handling, and comprehensive unit testing. These updates align with industry best practices for robust database solutions and enhance the overall user experience.

Full Documentation

Key Improvements

  • Error Handling: Structured exception handling was implemented in CRUD methods to improve debugging and ensure application reliability.
  • Query Optimization: MongoDB indexes were added for commonly queried fields (e.g., breed, location) to enhance performance and scalability.
  • Secure Credential Storage: Hardcoded credentials were replaced with environment variables for secure authentication and easier management.
  • Documentation: Inline comments and a detailed guide for MongoDB queries and CRUD operations were added to improve maintainability.
  • Unit Testing: Comprehensive unit tests were developed to validate create, read, update, and delete operations, ensuring database reliability.

Technical Implementation

The following technical changes were made as part of this enhancement:

  • Refactored AnimalShelter Class: Added error handling to log database exceptions and prevent runtime failures.
  • Created MongoDB Indexes: Optimized commonly queried fields, reducing response times and improving dashboard performance.
  • Credential Management: Replaced hardcoded credentials with environment variables using Python’s os.getenv(), enhancing security and maintainability.
  • Documentation: Documented MongoDB query structures and CRUD operations to assist future developers and simplify onboarding.
  • Unit Testing: Designed unit tests to validate CRUD functionality, using a separate test collection to avoid interfering with production data.
